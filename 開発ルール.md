# 開発ルール

## バージョン管理

- コードを変更するたびに `APP_VERSION`（`src/App.tsx`）と `package.json` の `version` を更新する。
- セマンティックバージョニング（`MAJOR.MINOR.PATCH`）を採用する。
  - **PATCH**: バグ修正、軽微な調整
  - **MINOR**: 新機能追加、UI改善
  - **MAJOR**: 破壊的変更、大規模リファクタ
- コミットメッセージに新バージョンを含める。例: `v0.2.0: Add Excel import/export`

## コミット・デプロイ

- コード変更後は必ず `npm run build` でビルド確認
- ビルド成功後に `git add -A && git commit && git push`
- GitHub Actions で GitHub Pages に自動デプロイされる
